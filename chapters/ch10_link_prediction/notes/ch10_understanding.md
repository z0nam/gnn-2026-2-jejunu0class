## 1. 출발점: 링크 예측이 노드 분류와 본질적으로 다른 이유

### 1.1 anchored vs unanchored

- 노드 분류: 모든 노드에 같은 함수 $f(\mathbf{z}_i)$를 적용. **"누구든 들어오면 라벨을 매겨라"**.
- 링크 예측: **"이 두 노드 $u, v$에 한해서"** 점수를 매겨라. 두 노드가 *특별한 위치*를 차지함.

수학적으로 노드 분류는 **노드 순열에 불변(equivariant)**이지만, 링크 예측은 **두 노드를 anchor로 지정한 후 그 anchor에 대해 비대칭적인 계산**이 필요합니다. 이게 표준 GNN과 잘 안 맞는 첫 이유.

### 1.2 데이터 분할의 단위

- 노드 분류: 노드를 train/val/test로 마스킹. 그래프 구조는 그대로.
- 링크 예측: **엣지**를 train/val/test로 분할. 게다가:
  - test 엣지는 메시지 패싱 그래프에서도 빼야 함 (안 빼면 누설)
  - 음성 엣지(없는 연결)를 무작위로 샘플링 → 매번 다른 음성

PyG의 `RandomLinkSplit`이 자동으로 처리하지만, **무슨 일이 일어나고 있는지** 이해하는 게 중요. "엣지가 학습 데이터이자 메시지 패싱 그래프의 일부"라는 이중성이 모든 누설 위험의 원천.

### 1.3 클래스 불균형의 구조적 성격

$N$개 노드 그래프에서:
- 가능한 모든 쌍의 수: $O(N^2)$
- 실제 엣지 수: 보통 $O(N)$ 정도 (희소 그래프)

→ 양성:음성 = $1:N$ 수준의 극단적 불균형. 그래서 학습/평가 모두 **음성 샘플링이 필수**고, 어떻게 샘플링하느냐가 성능 측정에 결정적 영향.

## 2. VGAE 깊이 읽기

### 2.1 GAE의 첫 발걸음: "인코딩 + 내적 디코딩"

가장 단순한 출발:

$$\mathbf{Z} = \text{GCN}(X, A), \quad \hat{A}_{uv} = \sigma(\mathbf{z}_u^\top \mathbf{z}_v)$$

이 한 줄이 담은 가설:
> *"두 노드가 임베딩 공간에서 가까우면 연결되어 있을 가능성이 높다"*

이건 정말로 **homophily 가설**입니다. 친구가 친구를 만든다, 비슷한 단어를 쓰는 논문이 서로 인용한다. Cora 같은 인용 네트워크엔 완벽히 들어맞음. 하지만:

- **이성성(heterophily) 그래프**: 이질적인 노드끼리 연결되는 경우 (예: 사기 탐지에서 사기범과 정상 사용자 사이 엣지, 단백질 결합에서 상보적 구조)
- **방향성**: $\sigma(\mathbf{z}_u^\top \mathbf{z}_v) = \sigma(\mathbf{z}_v^\top \mathbf{z}_u)$ → 항상 대칭. 디렉티드 그래프 표현 불가
- **글로벌 구조**: 내적은 두 점 사이의 관계만 표현, 더 큰 구조적 패턴은 무시

**핵심**: GAE의 한계는 "디코더가 너무 단순함"이 아니라, **"디코더의 형태가 강한 가정을 강제함"**입니다.

### 2.2 V를 붙이는 이유: 확률적 임베딩

VGAE는 임베딩을 **점**이 아니라 **가우시안 분포**로 봅니다:

$$q(\mathbf{z}_i \mid X, A) = \mathcal{N}(\bm{\mu}_i, \text{diag}(\bm{\sigma}_i^2))$$

두 GCN으로 $\bm{\mu}$, $\log \bm{\sigma}$를 각각 출력. 표본은 reparameterization trick:

$$\mathbf{z} = \bm{\mu} + \bm{\sigma} \odot \bm{\epsilon}, \quad \bm{\epsilon} \sim \mathcal{N}(0, I)$$

**왜 굳이 확률적인가?**

이론적 답: 변분추론(VI)의 ELBO 최소화 형태로 학습을 정당화. KL 항이 정규화 효과를 주어 임베딩이 너무 자유로워지는 걸 막음.

실용적 답: **솔직히 Cora 같은 벤치마크에서 GAE와 VGAE 성능 차이는 별로 안 큼**. 변분 부분의 진짜 가치는:
1. 임베딩에 노이즈를 주입하는 정규화 효과
2. (드물게) 임베딩의 불확실성을 표현 가능 — "이 노드는 어디 있는지 잘 모르겠다"

발제 메시지: *"V는 본질이라기보다 정규화 트릭에 가깝다. KL을 1/N로 나눈다는 사실 자체가 그 가중치를 약하게 보고 있다는 신호."*

### 2.3 손실 함수의 의미

$$\mathcal{L}_{\text{VGAE}} = \mathcal{L}_{\text{recon}} + \frac{1}{N} \text{KL}[q(Z|X,A) \,\|\, p(Z)]$$

- `recon_loss`: 양성 엣지는 $\hat{A}_{uv} \to 1$, 음성 엣지는 $\hat{A}_{uv} \to 0$이 되도록 BCE
- KL: 임베딩 분포를 표준정규로 끌어당김

**1/N의 의미를 정확히**:
- 표준 ELBO엔 1/N이 없음
- $N$개 노드 → KL은 노드 수에 비례해 누적 → 노드당 평균을 취하는 것이 1/N
- β-VAE 관점: KL 가중치를 작게 두어 reconstruction을 우선시 (posterior collapse 방지)

발제에서 받을 만한 질문: *"왜 1/N이고 다른 값이 아닌가?"* → 답: 경험적 선택. 다른 가중치도 가능, 결과 비슷.

### 2.4 학습이 끝난 뒤 우리가 얻는 것

`z = model.encode(test_data.x, test_data.edge_index)` → $(N, 16)$ 행렬.

이걸로 임의의 $(u, v)$ 점수를 즉시:
$$\hat{A}_{uv} = \sigma(\mathbf{z}_u^\top \mathbf{z}_v)$$

**이 단순함이 VGAE의 핵심 장점**입니다. $O(N^2)$의 모든 쌍을 한 번의 행렬곱으로. 추천 시스템처럼 후보 수가 폭발적인 환경에서 결정적.

## 3. SEAL 깊이 읽기

### 3.1 패러다임 전환: 표현 학습 → 문제 환원

GAE/VGAE의 사고: **"좋은 노드 표현을 학습하자, 그러면 링크는 표현의 함수로 풀린다"**.

SEAL의 사고: **"링크 예측 문제를 다른 문제로 환원하자: 서브그래프 이진 분류"**.

이건 단순한 모델 차이가 아니라 **문제 정의의 차이**입니다. 그래서 후속 작업도 다 달라짐.

### 3.2 왜 enclosing subgraph인가?

핵심 통찰: **모든 휴리스틱(CN, Adamic-Adar, Katz)이 결국 enclosing subgraph의 함수**라는 관찰.

- Common Neighbors: 1-hop enclosing subgraph만 보면 됨
- Adamic-Adar: 마찬가지로 1-hop
- Katz: $k$-hop으로 가면 모두 계산 가능

따라서:
> *"어떤 휴리스틱을 쓸지 미리 정하지 말고, $k$-hop enclosing subgraph 전체를 GNN에 주고 GNN이 알아서 최적의 함수를 학습하게 하자"*

이게 SEAL의 출발점입니다. 휴리스틱들의 **상위 집합**.

**γ-decaying heuristic 정리** (Zhang & Chen, 2018): 어떤 휴리스틱이 거리에 따라 지수적으로 감쇠한다면, 충분히 큰 $k$-hop enclosing subgraph만 봐도 임의의 정밀도로 근사 가능. **이게 SEAL의 표현력에 대한 이론적 보장**.

### 3.3 DRNL을 왜 발명했나 (이 챕터의 가장 깊은 부분)

#### 1-WL 한계의 진짜 의미

표준 GNN의 표현력은 **1-Weisfeiler-Lehman test**의 한계와 같다 (Xu et al. 2019).

- 1-WL: 노드 색깔을 "내 색 + 이웃 색의 집합"의 해시로 반복 갱신
- 두 노드가 1-WL로 같은 색이면 → GNN으로도 같은 임베딩

**결과**: 정규 그래프(모든 노드 차수 같음)에서 모든 노드의 임베딩이 동일. 임의의 두 쌍에 대해 같은 점수 → 링크 예측 표현력 0.

#### 구체적 예: 4-cycle

```
1 -- 2
|    |
4 -- 3
```

모든 노드 차수 2, 모든 이웃 모양 동일. 노드 feature가 같다고 가정하면:
$$\mathbf{z}_1 = \mathbf{z}_2 = \mathbf{z}_3 = \mathbf{z}_4$$

내적 디코더: $\sigma(\mathbf{z}_1^\top \mathbf{z}_2) = \sigma(\mathbf{z}_1^\top \mathbf{z}_3)$. 하지만 $(1,2)$는 엣지, $(1,3)$은 비엣지. **GNN이 구분 못 함**.

이건 학습 부족이 아니라 **모델 표현력의 구조적 한계**입니다.

#### Labeling Trick의 본질

처방: **노드 feature에 "타깃이 누구인지"를 박아넣어라**.

- Naive 방식: 타깃 노드만 1, 나머지 0의 one-hot
- DRNL: 타깃까지의 거리 $(d_u, d_v)$를 정수로 인코딩
- 더 일반: Distance Encoding, PGNN, ID-GNN 등

Zhang et al. 2021의 정리: **어떤 구조적 라벨이든 타깃 anchored 정보를 추가하면 GNN의 멀티노드 표현력이 회복된다**.

#### DRNL 공식 음미

$$z(x) = 1 + \min(d_u, d_v) + \frac{d}{2}\left(\frac{d}{2} + (d \bmod 2) - 1\right)$$

여기서 $d = d_u + d_v$.

왜 이 형태?
- $(d_u, d_v)$ 두 정수를 **단일 정수**로 매핑 → one-hot 임베딩이 간단
- 거리 합과 최솟값을 인자로 → 같은 합이면 분기, 다른 합이면 다른 영역
- "거리 클수록 z 크다" 단조성 유지

발제에서 굳이 공식을 외우게 할 필요는 없음. **"공통 이웃은 z=2, 양쪽 가까운 점은 z=3-4, 먼 점은 z=7"** 정도의 직관이면 충분.

### 3.4 누설(leakage) 방지의 디테일들

SEAL 코드를 읽을 때 가장 헷갈리는 부분이지만, 가장 중요한 디테일.

#### 디테일 1: 타깃 엣지 제거

```python
mask1 = (sub_edge_index[0] != src) | (sub_edge_index[1] != dst)
mask2 = (sub_edge_index[0] != dst) | (sub_edge_index[1] != src)
sub_edge_index = sub_edge_index[:, mask1 & mask2]
```

타깃 $(u,v)$의 엣지를 서브그래프에서 양방향 제거. **안 하면**: 학습 시 모델이 "엣지가 직접 보이네 → 1"로 trivial하게 학습. 양성 100% 맞히지만 음성 학습 신호가 없음 → 일반화 0.

#### 디테일 2: 거리 계산 시 상대 노드 제외

```python
d_src = shortest_path(adj_wo_dst, ..., indices=src)  # dst 제거 후
d_dst = shortest_path(adj_wo_src, ..., indices=dst)  # src 제거 후
```

**왜 이렇게?** 만약 dst를 그대로 두고 src로부터 거리를 재면, 많은 노드의 거리가 "dst를 거쳐가는 경로"로 짧아짐. 결국 d_src + d_dst가 항상 작은 값이 되어 라벨이 다 비슷해짐.

상대 노드를 제외함으로써 **"진짜 우회 경로"의 길이**를 측정. DRNL이 노드의 구조적 위치를 정확히 인코딩하게 됨.

#### 디테일 3: 양성/음성 같은 절차

이건 코드에선 자연스럽게 처리되지만 의식해야 함: 양성 쌍과 음성 쌍 모두 동일한 절차로 서브그래프를 만들어야 분포가 일치. 양성에만 타깃 제거를 하고 음성엔 안 하면 → 분포 차이로 trivial 분류.

### 3.5 DGCNN: 왜 굳이 sort pooling + Conv1D?

서브그래프 크기가 매번 다른데 → 고정 길이 표현이 필요. 보통 그래프 분류에선 sum/mean pooling을 쓰지만 SEAL은 **SortPooling**:

1. 마지막 GCN 레이어 출력의 **마지막 채널**로 노드를 정렬
2. 상위 $k=30$개만 선택 (부족하면 0-padding)
3. 정렬된 시퀀스를 Conv1D로 처리

**왜 sort?**
- Sum/mean은 "노드 순서 정보"를 완전히 버림 → 구조 표현력 손실
- Sort는 노드 간 "상대적 중요도"를 일관된 순서로 매김 → 텍스트처럼 처리 가능

**왜 Conv1D?**
- 정렬된 시퀀스에서 인접한 노드들은 비슷한 중요도 → 1D 합성곱이 잘 잡아냄
- 사실상 SEAL은 그래프를 시퀀스로 환원해 처리

DGCNN은 SEAL과 독립적인 모델이지만, "그래프 → 시퀀스 → CNN"이라는 발상이 SEAL의 패러다임(문제 환원)과 잘 어울림.

## 4. 두 모델의 진짜 차이 (핵심 메시지)

### 4.1 Transductive vs Inductive

| | VGAE | SEAL |
|---|---|---|
| 학습 단위 | 그래프 1개 | 후보 링크별 서브그래프 |
| 추론 시 새 노드 | 재학습 또는 별도 기법 | 그대로 적용 가능 |
| 새 그래프 일반화 | 어려움 | 자연스러움 |

**Cora 같은 closed-world 벤치마크에선 이 차이가 안 보이지만, 실무에선 결정적**:
- 추천 시스템에서 새 사용자가 등장: VGAE는 매번 재학습? SEAL은 학습된 모델로 즉시 추론
- 단백질 그래프에서 새 단백질: SEAL이 본질적으로 적합

### 4.2 Capacity-Efficiency 트레이드오프

VGAE:
- 표현력 제한 (내적, 대칭, homophily) ← 강한 가정
- 효율 좋음 (한 번 encode, 모든 쌍 즉시)

SEAL:
- 표현력 크다 (γ-decaying 휴리스틱 모두 포함)
- 효율 나쁨 (쌍마다 전처리)

**언제 무엇이 무너지나**:
- VGAE 무너지는 곳: 노드 feature가 약한 그래프, heterophily 그래프, 디렉티드 그래프
- SEAL 무너지는 곳: 대규모 그래프 (수억 쌍 평가 불가능), 노드 feature가 매우 중요한 도메인

### 4.3 Cora가 두 모델 모두에게 후한 이유

- **호모필리 강함**: 인용 네트워크는 비슷한 분야끼리 연결 → VGAE 내적 디코더 가정과 일치
- **노드 feature 풍부**: 1433-d BoW가 라벨링 역할 대신 → VGAE도 작동
- **그래프 작음**: SEAL의 전처리 비용 감당 가능
- **무방향**: 두 모델 모두 자연스럽게 처리

결과적으로 두 모델 모두 AUC ~0.87-0.89를 찍어서 **"둘이 비슷한 모델"처럼 보이는 착시**가 생김. 실제로는 매우 다른 가정/구조의 모델인데.

## 5. 본 실행 결과 해석

| 모델 | 책 | 본 실행 (CPU, 시드 0) |
|---|---|---|
| VGAE | AUC 0.88 / AP 0.90 | **0.87 / 0.87** ✓ |
| SEAL | AUC 0.89 / AP 0.90 | **0.79 / 0.80** (↓) |

VGAE는 거의 일치. SEAL이 낮게 나온 가능한 이유:
1. 30 epoch은 DGCNN에 짧음 (더 돌리면 회복 가능성)
2. 책 코드의 `BCEWithLogitsLoss` + `.sigmoid()` 출력 조합 — sigmoid가 두 번 적용되는 효과 (책의 알려진 이슈)
3. 음성 샘플링이 매 epoch 무작위 → 시드 의존
4. SortPooling의 $k=30$이 Cora 서브그래프에 비해 작을 수 있음

발제에서 *"본 실행이 책보다 낮은 이유"*를 솔직히 다루면 청중에게 "재현성"이라는 메타 메시지도 줄 수 있음.

## 6. 발제 메시지 후보

### 한 줄 핵심
> *"링크 예측은 GNN이 자연스럽게 풀 수 없는 anchored 과제다. VGAE는 노드 feature에 그 일을 떠넘기고, SEAL은 라벨링으로 직접 해결한다."*

### 청중에게 던질 질문 (Q&A용)

1. **표현력 vs 효율**: "당신의 도메인에서 노드 수와 후보 쌍 수가 1억을 넘는다면, 어느 모델을 쓸 것인가? 둘 다 안 된다면 그 다음 선택지는?"
2. **이성성**: "사기 탐지처럼 사기범과 정상 사용자 사이의 엣지를 예측해야 한다면 VGAE의 내적 디코더는 어디서 무너지는가?"
3. **DRNL 일반화**: "DRNL은 두 노드 라벨링 트릭. 만약 *세 노드 사이의 관계*(예: 삼각형 닫힘 예측)를 풀어야 한다면 어떻게 일반화할까?"
4. **누설**: "타깃 엣지를 서브그래프에서 제거하지 않으면 어떤 일이 일어나는가? 학습 AUC와 테스트 AUC 차이로 진단할 수 있나?"

### 슬라이드에서 강조할 그림 우선순위

1. **SEAL 파이프라인** (`ch10_fig03_seal_pipeline.png`) — 챕터의 플래그십. 청중이 한 그림만 기억해야 한다면 이것.
2. **Ahat 히트맵** (`ch10_runtime_vgae_ahat_heatmap.png`) — VGAE가 *블록 구조*를 학습한다는 걸 시각적으로 증명. "homophily를 학습한다"의 가장 강력한 증거.
3. **DRNL 실제 예시** (`ch10_runtime_drnl_example.png`) — z=2 공통 이웃이 보이는 게 핵심. DRNL이 왜 휴리스틱을 일반화하는지 한 그림에 담김.

현재 슬라이드는 책 흐름에 충실. 다음을 추가하면 발제의 깊이가 한 단계 올라감:

### 추가 권장 슬라이드

1. **"왜 GNN으로 링크 예측이 어려운가" 섹션 신설** (Section 1과 SEAL 사이)

   - 1-WL 한계 한 슬라이드

   - 현재 슬라이드의 비교표에 한 줄 추가 또는 별도 슬라이드
